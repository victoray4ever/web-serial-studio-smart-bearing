一、总体依据

1.1 设计总要求

面向嵌入式设备在线调试、工业传感数据可视化和远程监测场景，设计并实现一套基于浏览器运行的 Web Serial Studio Smart Bearing 网页监测系统。系统以静态网页部署为基础，以多协议数据接入、实时解析、动态可视化和项目化配置为核心，构建从数据采集、传输、解析到展示的完整链路，实现串口、WebSocket 与 MQTT 数据的统一接入以及面向 STM32 轴承监测数据的专用解析与显示。

本系统不依赖后端业务服务，前端可直接部署在 GitHub Pages 等静态托管平台，通过浏览器完成设备连接、项目加载、数据监测、参数查看和调试分析等功能。系统在结构设计上兼顾轻量化部署、模块化扩展和工程可维护性，满足实验室验证、现场调试、演示展示和协作开发等使用需求。

1.2 功能设计

本网页监测系统具备设备连接、数据解析、实时显示和配置部署四大功能模块，形成面向传感监测与调试场景的完整软件体系。

（1）多协议接入功能

系统支持 Serial、WebSocket 和 MQTT 三类通信方式。串口模式面向本地浏览器直连 MCU 或 USB 转串口设备；WebSocket 模式适用于局域网或云端转发场景；MQTT 模式适用于通过 Broker 进行发布订阅的数据接入场景。针对浏览器环境的限制，MQTT 接入明确采用 ws 或 wss 方式，避免将传统 TCP 端口误用于网页端连接。

（2）数据解析功能

系统支持 Quick Plot、Device Sends JSON、Project File 和 STM32 Binary 等解析模式。对于通用 CSV 数据，系统可直接按逗号分隔内容进行快速绘图；对于 JSON 自描述设备，系统可根据设备发送的布局描述自动生成界面；对于工程化项目文件，系统可依据预设 JSON 配置组织图表和仪表盘；对于当前 STM32 轴承监测场景，系统已实现专用二进制帧解析，可直接提取加速度、三路应变及两路温度数据。

（3）可视化显示功能

系统支持折线图、组合图、仪表盘、条形图、指南针、加速度组件和数据表等多种控件。仪表盘用于显示温度等慢变量，折线图用于观察高频加速度与应变波形，数据表用于同步显示当前值、最小值与最大值。系统支持自动布局、数据重置、滚动浏览以及缩放状态下纵坐标自适应，提升工程调试的可读性和交互体验。

（4）项目配置与部署功能

系统支持加载工程项目文件，自定义控件类型、数据索引、量程范围和显示单位。用户可基于项目文件快速切换不同监测任务，也可直接在浏览器端运行，不需要单独构建后端。项目同时集成 GitHub Pages 自动部署流程，便于将网页发布为在线可访问版本。

1.3 性能指标

本系统性能指标以网页端实时监测和前端部署使用为基础进行设定，主要用于评估系统的接入能力、解析能力、显示能力和工程应用能力。主要指标包括：

- 支持 Serial、WebSocket、MQTT 三类接入方式；
- 支持静态网页部署，无需后端服务；
- 支持 STM32 专用二进制数据流解析；
- 支持图表、仪表盘和数据表混合显示；
- 支持项目文件驱动的界面生成与量程配置；
- 支持 GitHub Pages 在线发布与浏览器访问。

各项性能指标说明如下：

（1）接入能力

系统可在浏览器安全上下文中完成本地串口连接，并可通过 WebSocket 或 MQTT over WebSocket 接入远程数据源，满足本地调试与远程监测两类典型需求。

（2）解析能力

系统支持文本帧和二进制帧两类数据格式，对于当前 STM32 轴承监测协议，可完成 5728 字节有效载荷级别的专用解析，并输出对应通道数据用于实时显示。

（3）显示能力

系统支持对高频波形和低频状态量进行并行显示，具备自动布局、滚动浏览、局部缩放和自适应纵轴等可视化功能，可满足实验调试和结果展示需求。

（4）工程应用性

系统采用原生 JavaScript ES Modules 开发，结构清晰、依赖简单、部署成本低，可作为后续更多监测项目、教学演示项目和实验平台前端界面的基础框架。

二、网页系统核心技术架构

本系统采用浏览器端静态前端架构，由状态管理、通信驱动、帧解析、界面布局和可视化控件五个层次组成。状态管理层负责维护连接模式、项目配置、总线类型和界面状态；通信驱动层负责与 Serial、WebSocket、MQTT 三类总线建立连接并接收数据；帧解析层负责识别文本帧或二进制帧并输出结构化数据；界面布局层负责根据默认布局或项目文件创建仪表盘控件；可视化控件层负责绘制图表、仪表盘和数据表。

在当前 STM32 轴承监测任务中，数据流总体路径为：下位机传感器采集数据后，通过串口或 MQTT 发布二进制帧；网页端 ConnectionManager 选择相应驱动接收数据；FrameParser 根据模式识别帧头、帧尾和有效载荷后提取数据；Dashboard 根据项目配置将加速度、应变和温度分别映射到 Plot、Gauge 与 DataGrid 组件中，实现实时显示。

三、网页系统结构设计

3.1 设计方案

3.1.1 总体设计

系统总体采用单页应用结构，以 `index.html` 作为入口页面，通过 ES Modules 分模块加载 `src` 目录中的核心逻辑。整个界面主要由顶部工具栏、左侧设备设置栏和中间仪表盘区域组成。顶部工具栏承担项目打开、保存、导出和接口切换等功能；左侧设备设置栏负责选择解析模式、连接方式和协议参数；中部仪表盘区域用于展示图表、仪表盘和数据表等可视化结果。

系统在设计上突出以下特点：

- 采用无后端静态部署结构，便于直接发布；
- 采用事件总线驱动模块通信，降低模块耦合；
- 采用项目文件描述监测布局，提升复用性；
- 采用专用 STM32 二进制解析逻辑，兼顾效率与稳定性；
- 采用可扩展控件结构，便于新增图表或传感器组件。

3.1.2 主要技术参数

系统主要技术参数如下：

- 前端技术栈：HTML、CSS、Vanilla JavaScript、ES Modules；
- 图表绘制：Chart.js；
- MQTT 接入：mqtt.js 浏览器版本；
- 支持总线：Serial、WebSocket、MQTT；
- 支持解析模式：Quick Plot、Device Sends JSON、Project File、STM32 Binary；
- 支持控件：Plot、Gauge、Bar、Compass、Accel、DataGrid；
- 部署方式：本地 HTTP 服务、GitHub Pages。

3.2 系统结构组成

本网页系统主要包括入口与全局状态模块、通信与连接模块、数据解析模块、界面与控件模块、项目配置模块和部署模块。

3.2.1 状态管理与事件机制设计

系统通过 `AppState.js` 统一维护操作模式、连接状态、总线类型、串口配置、WebSocket 配置、MQTT 配置、帧设置和项目对象等信息。通过 `EventBus.js` 在模块间分发状态变更、帧接收、Toast 提示和界面操作事件，避免直接跨模块引用带来的耦合问题。该机制使得工具栏、侧栏、连接管理器与仪表盘可以在松耦合前提下完成协同工作。

3.2.2 连接与通信设计

系统的通信层由 `ConnectionManager.js` 统一管理，根据当前总线类型选择 `SerialDriver.js`、`WebSocketDriver.js` 或 `MqttDriver.js` 建立连接。连接成功后，驱动将原始数据以 `Uint8Array` 形式传递给 `FrameParser`。其中 MQTT 模块针对浏览器环境进行了专门约束，要求使用 WebSocket 端点，并支持 MQTT 3.1、3.1.1 和 5.0 配置。同时在侧栏中增加了 Broker URL 预览与错误提示，便于用户定位端口、路径和协议不匹配等问题。

3.2.3 数据解析设计

`FrameParser.js` 是本项目的核心数据处理模块。其一方面支持普通 CSV 和 JSON 分帧，另一方面针对 STM32 轴承监测协议实现了专用二进制帧解析逻辑。当前协议中，加速度波形按有符号 ADC 原始值乘以 0.000488 换算为 g，并对 Z 轴静止重力分量进行修正；三路应变与一路 ADC 温度按有符号原始值乘以 2.5 再除以 8388608.0 进行换算；TMP117 温度按原始值乘以 0.0078125 进行换算。

系统在 MQTT 场景下考虑到消息边界、十六进制 ASCII 负载和原始二进制负载的差异，增加了直接载荷解析、载荷流对齐与长零区识别等机制，以提高非标准分包情况下的解析稳定性。

3.2.4 可视化与交互设计

`Dashboard.js` 根据默认布局或项目文件动态创建控件实例，并将控件挂载至仪表盘区域。当前项目采用总览折线图加温度仪表盘加全量数据表的组合方式展示轴承监测信息。为提升使用体验，系统对滚动机制进行了修正，使仪表盘区域在卡片较多时可正常向下浏览；同时在折线图缩放和平移过程中加入了纵坐标按当前可视窗口自适应重算的机制，使放大后的小幅波动更清晰可辨。

3.2.5 项目配置与部署设计

系统通过 `projects/stm32-bearing.json` 描述监测页面，包括数据索引、标题、单位、量程和控件类型。当前配置中，将加速度通道设置为 `Accel Z`，三路应变使用 `MultiPlot` 统一显示，两路温度使用 `Gauges` 显示，并在底部使用 `DataGrid` 展示综合数据。部署方面，项目提供 GitHub Actions 工作流，在推送到目标分支后可通过 GitHub Pages 进行静态发布，实现在线访问。

四、系统调试与验证

4.1 本地运行验证

项目采用 ES Modules 组织代码，开发阶段需通过本地 HTTP 服务运行，例如使用 `python -m http.server 8000` 启动服务，再在浏览器中访问 `http://localhost:8000`。经本地调试验证，系统能够正常加载项目文件、完成串口连接、显示 STM32 监测数据，并支持页面滚动、图表缩放和数据导出。

4.2 MQTT 调试验证

在 MQTT 调试过程中，已验证浏览器端必须连接 Broker 暴露的 ws 或 wss 地址，而不能直接连接原始 TCP 端口；在 HTTPS 页面下必须使用 wss；同时需确认 WebSocket 路径、topic 和协议版本配置正确。系统已在界面中增加端点预览和更明确的连接错误提示，用于指导用户快速定位连接失败原因。

4.3 解析结果验证

结合 STM32 实际采集数据和导出的 CSV 结果，系统已完成当前协议的解析修正。调试后加速度通道在静止状态下可呈现接近 0 g 的修正结果，应变与温度通道可按设定公式输出工程量。界面中的仪表盘、折线图与数据表显示结果保持一致，满足当前实验需求。

五、结论与后续规划

本设计说明书围绕 Web Serial Studio Smart Bearing 网页监测系统的总体要求、技术架构、模块设计、解析逻辑、交互优化和调试验证进行了系统说明。项目已具备浏览器端多协议接入、STM32 二进制协议解析、实时可视化显示和静态网页部署能力，能够满足当前轴承监测实验与在线展示需求。
